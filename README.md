# 离线书架

一个移动端优先的 私人阅读库。它可以保存作品、备注、文件夹、标签、章节、阅读进度、书签和高亮。数据存在当前浏览器本机数据库里。

## 手机使用方式

最适合手机的导入方式是下载原站的 HTML 文件：

1. 用手机浏览器打开作品页。
2. 点页面上的 `Download`。
3. 选择 `HTML`。
4. 回到离线书架网页，点 `导入 HTML 文件`。
5. 选择刚下载的 `.html` 文件。

这样可以绕开浏览器跨站限制，也更适合长篇、多章节作品。

## 电脑抓取后同步到手机

当前已经支持手动同步：

1. 电脑端导入作品。
2. 点 `导出书架`，得到一个 `reading-vault-日期.json` 文件。
3. 把这个 JSON 文件发到手机，比如微信文件、iCloud、网盘、AirDrop 或数据线。
4. 手机打开书架网页，点 `导入书架`。

导入后，正文、章节、tag、备注、文件夹、阅读进度、书签和高亮都会在手机本地保存。

## GitHub Pages 打开方式

把这个项目上传到 GitHub 仓库后，启用 GitHub Pages。仓库里已经带了 `.github/workflows/pages.yml`，推送到 `main` 分支后会自动把 `public` 文件夹发布成网页。

发布成功后，GitHub 会给你一个类似这样的地址：

```text
https://你的用户名.github.io/仓库名/
```

手机和电脑都可以直接打开这个地址。手机浏览器里还可以选择“添加到主屏幕”，用起来会更像一个小 App。

## 自动同步后端方案

如果想要“电脑导入后手机自动出现”，需要一个小后端。推荐结构：

- GitHub Pages：只放网页界面。
- 后端服务：负责抓 原站、保存你的书架数据、给手机同步。
- 数据库：SQLite 或 Postgres。
- 登录方式：先做一个私人同步密码即可。

适合部署的平台：

- Render
- Railway
- Fly.io
- 自己的服务器或 NAS

后端需要提供这些接口：

```text
POST /api/import      粘贴 原站链接后抓取正文
GET  /api/library     手机/电脑读取书架
PUT  /api/library     上传最新书架
POST /api/backup      生成备份
```

## 本地打开

双击 `打开原站离线书架.cmd`，看到本地服务启动后，在浏览器打开：

```text
http://localhost:4175
```

本地版可以尝试自动读取 原站链接；如果 原站临时访问失败，仍然可以用 HTML 文件导入。

## 离线保存提醒

- 正文、标题、作者、tag、章节、备注、书签、高亮：保存在当前浏览器本地。
- 作者删文后，只要你没有清理该网页的站点数据，已经保存的正文还在。
- 图片目前会尽量保留，部分图片可能仍依赖原链接。后续可以做“下载图片到本地”的独立功能。
- 换手机、换浏览器、换网址时，本地数据不会自动过去，需要用 `导出书架 / 导入书架` 或同步后端。
